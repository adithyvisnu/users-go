# Users API

Multi-tenant CRUD API for user management, written in Go.

**Stack:** Gin · GORM · PostgreSQL · Redis (cache-aside) · Swaggo

---

## Project Structure

```
users-api/
├── cmd/api/main.go                  # Entrypoint + Swagger root annotations
├── config/config.go                 # Env-based configuration
├── docs/                            # Auto-generated by swag (do not edit)
├── internal/
│   ├── cache/cache.go               # Redis helpers + key builders
│   ├── handler/user_handler.go      # HTTP handlers with Swagger annotations
│   ├── middleware/auth.go           # JWT auth + tenant injection
│   ├── models/user.go               # GORM model + DTOs
│   ├── repository/user_repository.go # PostgreSQL + Redis cache-aside
│   └── service/user_service.go      # Business logic
├── docker-compose.yml
├── Dockerfile
└── Makefile
```

---

## Quick Start

### 1. Install tools

```bash
# Install swag CLI (needed to generate docs)
make install-tools
```

### 2. Run with Podman

```bash
# Install podman-compose if you don't have it
pip install podman-compose

make docker-up
# API: http://localhost:8080
# Swagger UI: http://localhost:8080/swagger/index.html
```

### 3. Run locally

```bash
# Requires Postgres + Redis running locally
make run
```

## API Endpoints

| Method | Path             | Description               |
|--------|-----------------|---------------------------|
| GET    | /health          | Health check              |
| GET    | /v1/users        | List users (paginated)    |
| POST   | /v1/users        | Create user               |
| GET    | /v1/users/:id    | Get user by ID            |
| PATCH  | /v1/users/:id    | Update user (partial)     |
| DELETE | /v1/users/:id    | Soft-delete user          |

All `/v1/*` routes require `Authorization: Bearer <token>`.

## Redis Caching Strategy

Every read goes through a **cache-aside** pattern:

```
Request → Check Redis → HIT: return cached value
                     → MISS: query PostgreSQL → store in Redis → return
```

**Cache keys:**
```
users-api:tenant:<tenantID>:user:<userID>          TTL: 10m  (single user)
users-api:tenant:<tenantID>:users:page:<p>:limit:<l>  TTL: 2m   (user lists)
```

**Invalidation on write:**
- `Create` / `Update` / `Delete` → delete single-user key + bust all list keys for that tenant (via `SCAN` + pattern delete).

---

## Multi-Tenancy

`tenant_id` is extracted from the JWT on every request and injected into the Gin context. The repository layer always scopes queries with `WHERE tenant_id = ?` — clients never supply the tenant ID directly.

## Environment Variables

| Variable       | Default                        | Description              |
|---------------|-------------------------------|--------------------------|
| `PORT`         | `8080`                         | HTTP port                |
| `DATABASE_URL` | `host=localhost ...`           | Postgres DSN             |
| `REDIS_ADDR`   | `localhost:6379`               | Redis address            |
| `REDIS_PASSWORD` | _(empty)_                    | Redis password           |
| `JWT_SECRET`   | _(required in production)_    | JWT signing key          |
| `ENV`          | `development`                  | `development`/`production` |
